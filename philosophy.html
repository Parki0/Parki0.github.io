Explain one thing at a time.

One tutorial should cover one and only one thing.
This is very important because if tutorial explains several things at a time that would be a waste of time of one who reads the tutorial.

People with different learning skill have to read the same article.

Some people can get things on the fly, some people need in depth explanations.
If you write a short version of tutorial for the first group, the second group won't understand that. And vise versa.
I think that this method could solve this problem.

Feedback.

Tutorials are mass product so if you write a good tutorial with some bottleneck which is hard to understand you will waste peoples time. So you need to ask for a feedback. And don't you tell me that there is a comment section where everyone can say that something is wrong. Because we don't tend to attempt to do thing we think will fail. To write a critic is a work, no one going to do that if they would know that you won't listen to them.
